Problem Statement

Understanding the challenge at Kibble Palace

The Challenge

The Kibble Palace, a heritage-listed 19th-century glasshouse in Glasgow Botanic Gardens, attracts a wide range of local and international visitors but currently lacks a reliable, accurate, and low-cost method to track visitor footfall.

Previous Attempts

While previous attempts such as cellphone signal tracking were explored, these proved inaccurate and unreliable for gathering meaningful visitor data.

Critical Need

Accurate visitor data — including metrics like peak times, seasonal trends, and basic demographics — is crucial for supporting budget allocation decisions, especially as city council funding continues to decline due to competing priorities like education.

Solution Requirements

Discreet implementation
High accuracy
Comprehensive metrics
Demographics capture
Cost-effective
Heritage site compliant

Proposed Solution

QR Code & Kiosk-Based Visitor Tracking

Hybrid Approach

We propose a dual-method solution that combines modern technology with accessibility considerations:

Voluntary QR Code Check-In

Smartphone users can quickly scan QR codes at the entrance for instant check-in/out

Discreet Kiosk Station

Alternative interface for visitors without smartphones or those preferring physical interaction

Visual Analytics Dashboard

Real-time data visualization for staff to monitor visitor patterns and trends

No Structural Modifications

Design Journey

Comprehensive documentation of our user-centered design process

User-Centric Problem Statement

Visitors to the Kibble Palace need a seamless, non-intrusive check-in process that’s accessible, multilingual, and respects their privacy. Meanwhile, stakeholders require accurate, timely, and low-cost visitor data to support budgeting and operational planning—all within the constraints of a heritage site.

Primary User Groups Identified

  • Tech-Savvy Tourists: International visitors comfortable with QR codes and mobile technology
  • Elderly Visitors: Visitors who prefer physical interfaces and may lack smartphones
  • Families with Children: Parents managing multiple children who need quick, simple check-in processes
  • Educational Groups: School trips and organized tours requiring group management
  • Stakeholder: Represents Kibble Palace operations, requiring accurate, discreet, and low-cost footfall data for budgeting and planning.

Core User Needs

  • Quick and effortless entry/exit process
  • No mandatory app downloads or registrations
  • Accessible options for all age groups and abilities
  • Privacy-respecting data collection
  • Clear instructions in multiple languages
  • Accurate visitor data to support budgeting and operational planning
  • Discreet and non-intrusive implementation
  • Minimal to no installation or hardware changes due to heritage restrictions
  • Low or zero implementation cost
  • Ability to scale for future enhancements (e.g., entry fees)

Prototyping Methods & Iterations

We followed a progressive prototyping approach, starting with low-fidelity concepts and gradually increasing detail and interactivity based on user feedback and technical validation.

01

Paper Prototypes

Week 1-2
Lo-Fi Paper Sketches
  • Hand-drawn interface layouts for both QR and kiosk systems
  • User flow diagrams showing check-in/check-out processes
  • Multiple layout variations for accessibility testing
  • Quick iteration cycles with immediate stakeholder feedback
QR Code Scanning Lo-Fi

Initial paper sketches exploring QR code scanning user flow and mobile interface layout

Kiosk Interface Lo-Fi

Hand-drawn kiosk interface concepts focusing on accessibility and large button design

Key Insights from Paper Prototyping:
  • Button sizes needed to be larger for elderly users
  • QR code placement critical for smartphone camera angles
  • Visual hierarchy important for quick comprehension
  • Multi-language support essential from the start
02

Digital Wireframes

Week 2-3
Figma Wireframes
  • Low-fidelity digital wireframes in Figma
  • Basic interaction flows and state changes
  • Responsive layouts for different screen sizes
  • Component library for consistent design elements
Wireframe Testing Results:
  • Simplified navigation reduced user confusion
  • Clear visual feedback improved task completion
  • Accessibility features validated with diverse user group
  • Information architecture refined based on user mental models
03

Interactive Mockups

Week 3-4
High-Fidelity Figma Prototypes
  • Fully interactive prototypes with realistic data
  • Micro-interactions and animation specifications
  • Complete user flows from entry to exit
  • Error states and edge case handling
User Testing Insights:
  • 95% task completion rate for QR check-in flow
  • 87% success rate for kiosk interface (elderly users)
  • Average completion time: 15 seconds (QR), 25 seconds (kiosk)
  • High satisfaction scores for visual design and clarity
04

Functional Demos

Week 4-5
Working HTML/CSS/JS Prototypes
  • Functional QR code generation and scanning simulation
  • Interactive kiosk interface with real-time updates
  • Basic analytics dashboard with live data visualization
  • Integration testing with Google Forms backend
Technical Validation:
  • QR codes work reliably across different smartphone cameras
  • Kiosk interface responsive to touch and accessibility tools
  • Data collection and storage meets GDPR requirements
  • System performance acceptable for expected visitor volumes

Project Reflection

Partner interactions, learnings, and honest assessment of our work

Interactions with Partners

Our collaboration with Kibble Palace stakeholders was instrumental in shaping our solution. Through regular meetings and feedback sessions, we gained valuable insights into the operational realities of managing a heritage site.

Key Partnership Learnings:

  • Heritage Constraints: Learned about strict limitations on structural modifications and the importance of preserving the building's historical integrity
  • Visitor Demographics: Discovered the diverse age range and technology comfort levels of typical visitors
  • Operational Needs: Understood staff workflows and the importance of real-time data for crowd management
  • Budget Realities: Gained appreciation for the financial constraints facing public institutions

The stakeholders' emphasis on inclusivity and accessibility fundamentally changed our approach from a purely digital solution to our hybrid QR/Kiosk system. Their feedback was crucial in validating our design decisions and ensuring practical feasibility.

What We're Proud Of (And What We're Not)

What We're Proud Of

Inclusive Design Approach: We're particularly proud of how we prioritized accessibility from the beginning. Rather than creating a solution that only works for tech-savvy users, we designed a system that truly serves all visitor demographics. The dual QR/Kiosk approach ensures no one is excluded from participating.

Stakeholder-Centered Process: Our commitment to continuous stakeholder engagement resulted in a solution that addresses real operational needs rather than theoretical problems. We listened, adapted, and iterated based on genuine feedback.

Heritage Sensitivity: We successfully balanced modern technology needs with respect for the historical significance of Kibble Palace. Our non-invasive approach preserves the building's integrity while providing valuable functionality.

Team Collaboration: Despite diverse backgrounds and expertise areas, our team maintained excellent communication and shared ownership of the project outcomes. Each member contributed meaningfully to the final solution.

Areas for Improvement

Technical Implementation Depth: While our prototypes effectively demonstrate the concept, we acknowledge that a production-ready system would require more robust backend infrastructure and security measures than we were able to fully develop within our timeframe.

User Testing Scale: Although we conducted user interviews and gathered feedback, we would have benefited from more extensive testing with actual Kibble Palace visitors to validate our assumptions about user behavior and preferences.

Data Analytics Sophistication: Our dashboard provides basic metrics, but we recognize that more advanced analytics capabilities (predictive modeling, trend analysis) would provide greater value to stakeholders.

Implementation Planning: While we focused heavily on design and user experience, we could have developed more detailed implementation timelines and change management strategies for actual deployment.

Key Learnings

This project taught us that successful design isn't just about creating elegant solutions—it's about understanding constraints, respecting context, and prioritizing user needs over technological sophistication. The most innovative solution isn't always the most complex one.

We learned the value of iterative design and the importance of remaining flexible as new information emerges. Our willingness to pivot from a mobile-only solution to a hybrid approach ultimately resulted in a much stronger final product.

User Story Board Flow

×

Our 6-Panel User Story Board

You can see our complete visitor journey storyboard: user approaches, scans QR code, completes form, receives confirmation, enjoys the Palace, then checks out. Each panel represents a key moment in the user experience.

1

🚶‍♀️ Visitor Approaches

What happens: A visitor arrives at Kibble Palace entrance and notices the tracking system

User goal: Enter the Palace and begin their visit

Design insight: The system must be immediately visible but non-intrusive to maintain the heritage site's atmosphere

2

📱 Scans QR Code

What happens: Visitor uses their smartphone to scan the QR code at the entrance

User goal: Quick and easy check-in process

Design insight: QR code placement and size must accommodate users of all ages and tech comfort levels

3

📝 Completes Form

What happens: Simple, optional form appears asking for basic demographic information

User goal: Provide helpful data while maintaining privacy

Design insight: Form must be minimal, mobile-friendly, and clearly explain data usage for GDPR compliance

4

✅ Receives Confirmation

What happens: System confirms check-in and provides welcome message or useful information

User goal: Feel welcomed and informed about their visit

Design insight: Confirmation should add value (opening hours, highlights, facilities) rather than just acknowledge completion

5

🌿 Enjoys the Palace

What happens: Visitor explores freely while system passively tracks visit duration and patterns

User goal: Immersive, uninterrupted experience in the beautiful greenhouse

Design insight: Tracking must be completely invisible during the visit - no notifications, prompts, or interruptions

6

👋 Checks Out

What happens: Optional check-out process or automatic detection when leaving

User goal: Smooth exit with optional feedback opportunity

Design insight: Exit should be frictionless with optional survey for those who want to provide feedback

Feature Prioritization Matrix

×

Our Feature Prioritization Strategy

This Impact vs Effort matrix helped us decide what to build first for our Kibble Palace visitor tracking prototype. We plotted each feature based on how much value it brings versus how difficult it is to implement.

🎯 Quick Wins (High Impact, Low Effort)

These became our MVP features - maximum value with minimal development time.

  • 📱 QR Code Check-in using Mobile Phone: Visitors love the simplicity, just scan and go!

💡 Why we prioritized this: Low cost, immediate deployment, and reliable data collection

🚀 Future Roadmap (High Impact, High Effort)

These features will transform the experience but need significant investment.

  • 📡 Bluetooth Beacon Tally Counter: Detects presence when visitors enter specific zones
  • 📶 Passive WiFi Signal Sensing: Track number of visitors based on mobile device connections
  • 🌡️ Heat Signature Counting with Thermal Cameras: Count the number of body signatures using thermal imaging cameras positioned discreetly

🔧 Nice-to-Haves (Low Impact, Low Effort)

Easy additions that enhance the basic tracking systems.

  • 👥 Volunteer "Human Sensor" Network: Train volunteers with handheld clicker/counter at peak times
  • 🚪 Entrance Mat Pressure Sensors: Use wireless pressure-sensitive mats at doorways, battery powered, reasonable stability
